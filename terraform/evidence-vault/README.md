# Evidence Vault

## Overview

This Terraform module creates a secured S3 evidence vault for storing generated GRC evidence artifacts.

The vault is designed for evidence retention, integrity, and controlled audit review.

It supports the broader AWS Continuous Compliance Automation Framework by providing a secure location for generated evidence, risk summaries, Terraform artifacts, and audit packages.

## Security Features

- S3 Object Lock enabled at bucket creation
- S3 versioning enabled
- Default server-side encryption
- Full public access block
- TLS-only bucket policy
- Bucket deletion protection policy
- Default retention period
- Machine-readable Terraform outputs

## Why This Matters

Compliance evidence should not live only on a local workstation.

A mature GRC Engineering workflow should support:

- Centralized evidence storage
- Tamper-resistant retention
- Evidence chain of custody
- Auditor or control owner access
- Evidence integrity validation
- Separation of source code from generated evidence

This evidence vault provides the retention and integrity layer for the framework.

## Object Lock Mode

| Mode | Use Case |
|---|---|
| GOVERNANCE | Lab and development use. Privileged users can bypass retention if explicitly allowed. |
| COMPLIANCE | Production evidence retention. Retention cannot be bypassed until expiration. |

Use `GOVERNANCE` for this project unless you intentionally want immutable evidence that cannot be deleted until the retention period expires.

## Control Alignment

| Control Theme | Implementation |
|---|---|
| Evidence Retention | S3 Object Lock and versioning |
| Evidence Integrity | SHA-256 manifest generated by capture script |
| Access Control | Private S3 bucket with full public access block |
| Encryption at Rest | Default S3 server-side encryption |
| Secure Transport | TLS-only bucket policy |
| Audit Traceability | S3 object VersionId recorded after upload |
| Chain of Custody | Evidence bundle includes hashes, commit metadata, and capture timestamp |

## Usage

From this directory:

```bash
terraform init
terraform validate
terraform plan -out=tfplan
terraform apply -auto-approve tfplan
```

Get the vault name:

```bash
terraform output -raw vault_name
```

View the vault attestation:

```bash
terraform output -json evidence_vault_attestation
```

## Outputs

| Output | Description |
|---|---|
| `vault_name` | S3 bucket name of the evidence vault |
| `vault_arn` | ARN of the evidence vault |
| `object_lock_mode` | Default Object Lock retention mode |
| `retention_days` | Default evidence retention period in days |
| `evidence_vault_attestation` | Machine-readable attestation for vault security controls |

## Evidence Publishing

Use the capture script from the project root:

```bash
RUN_ID="manual-$(date -u +%Y%m%dT%H%M%SZ)"
VAULT="$(terraform -chdir=terraform/evidence-vault output -raw vault_name)"

bash scripts/capture-evidence.sh \
  --workspace . \
  --run-id "$RUN_ID" \
  --vault "$VAULT" \
  --profile grc-engineer
```

The script uploads a bundled evidence package and prints a JSON receipt containing the S3 object key and VersionId.

Example receipt:

```json
{
  "run_id": "manual-20260507T120000Z",
  "vault": "aws-grc-grc-evidence-vault-abcd1234",
  "key": "runs/manual-20260507T120000Z/bundle.tar.gz",
  "version_id": "example-version-id",
  "captured_at_utc": "2026-05-07T12:00:00Z"
}
```

## Evidence Bundle Contents

The capture script may include:

- `evidence-results.json`
- `evidence-results.csv`
- `risk-summary.json`
- Terraform `.tf` files from the compliant S3 primitive
- `commit.txt`
- `python-version.txt`
- `terraform-version.txt`
- `manifest.json`

The `manifest.json` file contains SHA-256 hashes, file sizes, and capture timestamps for bundled artifacts.

## Verification

Verify Object Lock configuration:

```bash
VAULT="$(terraform output -raw vault_name)"

aws s3api get-object-lock-configuration \
  --bucket "$VAULT" \
  --profile grc-engineer
```

Verify bucket encryption:

```bash
aws s3api get-bucket-encryption \
  --bucket "$VAULT" \
  --profile grc-engineer
```

Verify public access block:

```bash
aws s3api get-public-access-block \
  --bucket "$VAULT" \
  --profile grc-engineer
```

Verify retention on an uploaded object:

```bash
aws s3api get-object-retention \
  --bucket "$VAULT" \
  --key runs/example-run-id/bundle.tar.gz \
  --profile grc-engineer
```

## Cleanup

For lab use with `GOVERNANCE` mode and short retention, wait until retention expires or delete with appropriate bypass permissions.

Destroy the Terraform-managed infrastructure:

```bash
terraform destroy -auto-approve
```

If the bucket contains retained object versions, Terraform may not be able to destroy the bucket until retention expires or protected versions are removed.

## Important Notes

Object Lock must be enabled when the bucket is created. It cannot be enabled later on an existing bucket.

Use `GOVERNANCE` mode for testing. Use `COMPLIANCE` mode only when you intentionally want evidence that cannot be deleted or bypassed until the retention period expires.

Do not store unsanitized public portfolio evidence in this vault unless the bucket is private, encrypted, access-controlled, and appropriate for the data sensitivity.
